# PQ-AI 架构摘要

## 核心数据链

```text
生产事件
  -> 五个工艺阶段实绩
  -> 程序版本 / 刷子表版本
  -> 刷子号参数
  -> 刷子到测量点贡献权重
  -> 点位工艺特征
  -> 橘皮 / 色差 / 光泽度 / 膜厚结果
  -> 预测 / 诊断 / 推荐
  -> 审批 / 执行 / 复测 / 效果评价
```

## 边界

- MySQL 是业务事实和审计记录的唯一主数据库。
- 原始文件、模型制品和训练数据快照应存放在对象存储，MySQL 保存元数据与血缘。
- 工艺推荐必须经过硬安全边界检查和人工审批，不允许首版自动修改机器人参数。
- 配置值和生产实绩值必须分开存储，AI 训练优先使用实绩值。
