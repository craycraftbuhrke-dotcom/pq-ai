# PQ-AI 开发计划

## 0. 执行纪律

- 开发必须先更新本计划，再按计划逐步实施，不允许跳步并行堆功能。
- 每完成一步，必须执行：代码复审、针对性测试、人工验证或浏览器验证，然后才能在本计划中标记完成。
- 基础建设顺序固定为：主数据与关系 -> 工艺阶段参数/设备参数采集 -> 质量数据采集 -> 测量点聚合 -> 各模块业务流程跑通 -> UI 基础治理 -> AI 模型与闭环深化。
- AI 模型、推荐、受控试验和闭环优化类工作排在基础数据链和基础业务流程稳定之后。
- 所有步骤都必须遵守项目规则和数据库规则：不得自动执行 MySQL DDL，不得使用物理外键，不得执行物理删除，不得绕开既有受治理服务链路。

## 1. 业务目标

PQ-AI 以“生产事件 + 测量点”为主线，将中涂、色漆、清漆三个涂层体系中的五个喷涂执行阶段、Dürr 机器人/旋杯/轨迹、材料实绩和漆膜质量结果关联起来，形成：

1. 质量预测：在质量测量前预测膜厚、橘皮、色差/效应风险。
2. 问题诊断：在排除仪器与数据质量问题后，定位对异常点位影响最大的工艺、材料和设备相关因素。
3. 参数推荐：在设备硬边界、材料 TDS 和工艺规则约束内生成调整建议。
4. 受控闭环：推荐经人工审批后生成程序版本，执行、复测并评价真实改善效果。

首版不允许 AI 直接修改机器人参数；所有推荐必须可解释、可审批、可追溯、可撤销。

明确排除预处理、电泳、涂胶、喷房温湿度、烘房温度、调漆间和光泽度。历史非批准业务数据入口与目录中的越界字段必须先隔离，再重建特征和模型。

## 2. 领域划分

### 主数据

- 工厂、车型、颜色、零件、测量编组、测量点。
- 颜色资料包含供应商、特征值、TDS、MSDS、COA、DOE 和数字标准。
- 点位必须关联零件、区域、质量指标类型和匹配点属性。

### 工艺程序

- 喷涂程序支持多个车型和多个颜色。
- 程序按版本管理，版本包含多个刷子表和刷子号参数。
- 刷子号与测量点通过贡献权重关联，用于处理喷幅重叠。
- 参数采用“参数定义 + 参数值”建模，覆盖五个执行阶段的流量、转速、内外成型空气、静电、距离、间距、速度等。
- 机器人、旋杯、控制器、钟罩、轨迹程序/校验和、TCP、路径段、姿态、触发和重叠必须版本化。

### 生产实绩

- 生产事件记录工厂、车型、颜色、批次、班次和时间。
- 每个生产事件包含五个喷涂执行阶段实绩、实际程序版本、材料批次和实际参数。
- 配置参数与生产实绩分开保存，AI 训练优先使用实际参数。

### 质量与标准

- 质量测量统一记录测量编号、编组、点位、数据类型、状态、时间和测量人。
- 橘皮、色差/效应和膜厚指标以标准化指标值存储，并关联适用标准。
- 测量必须关联仪器、探头/方法、固件/导入模板、校准/参考状态、重复测量和原始数据。
- 标准支持工厂、车型、颜色、零件、点位、数据类型和有效期等适用范围。

### AI 与闭环

- 点位特征快照保存一次预测所使用的完整工艺、材料和上下文特征。
- 模型版本记录训练数据血缘、指标、制品地址、上线状态和适用范围。
- 预测、诊断、推荐、审批、执行和复测评价全程留痕。
- 数据集按生产事件/车身/批次分组并按时间验证，禁止同一事件点位跨训练/验证集。
- 候选模型必须生成时间留出、生产分组留一和可用工厂/车型/颜色验证轴证据；缺少轴多样性必须明示，不得伪造跨域验证。
- 模型工件必须登记哈希，验收和激活时复核哈希一致性。
- 推荐审批前必须创建并批准受控试验计划，记录假设、证据类型、风险、回滚和持续观察方案。
- 解释只代表关联；推荐必须基于约束并经受控试验验证。

## 3. 核心数据链

```text
生产事件
  -> 三个涂层体系 / 五个执行阶段实绩
  -> Dürr 设备、程序、轨迹与实际参数
  -> 按目标族版本化的点位贡献
  -> 受治理的材料批次特性
  -> 点位特征快照
  -> 有仪器/方法/校准血缘的质量测量与标准判定
  -> 预测 / 诊断 / 推荐
  -> 人工审批 / 新程序版本
  -> 执行 / 复测 / 效果评价
```
