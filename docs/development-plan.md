# PQ-AI 开发计划

## 1. 业务目标

PQ-AI 以“生产事件 + 测量点”为主线，将五段喷涂工艺参数、刷子贡献权重、材料实绩和漆膜质量结果关联起来，形成：

1. 质量预测：在质量测量前预测膜厚、橘皮、色差、光泽度风险。
2. 问题诊断：定位对异常点位影响最大的工艺、材料、设备和环境因素。
3. 参数推荐：在设备硬边界、材料 TDS 和工艺规则约束内生成调整建议。
4. 受控闭环：推荐经人工审批后生成程序版本，执行、复测并评价真实改善效果。

首版不允许 AI 直接修改机器人参数；所有推荐必须可解释、可审批、可追溯、可撤销。

## 2. 领域划分

### 主数据

- 工厂、车型、颜色、零件、测量编组、测量点。
- 颜色资料包含供应商、特征值、TDS、MSDS、COA、DOE 和数字标准。
- 点位必须关联零件、区域、质量指标类型和匹配点属性。

### 工艺程序

- 喷涂程序支持多个车型和多个颜色。
- 程序按版本管理，版本包含多个刷子表和刷子号参数。
- 刷子号与测量点通过贡献权重关联，用于处理喷幅重叠。
- 参数采用“参数定义 + 参数值”建模，覆盖五段工艺的流量、转速、内外成型空气、静电、距离、间距、速度、粘度和固含比等。

### 生产实绩

- 生产事件记录工厂、车型、颜色、批次、班次和时间。
- 每个生产事件包含五个工艺阶段实绩、实际程序版本、材料批次和实际参数。
- 配置参数与生产实绩分开保存，AI 训练优先使用实际参数。

### 质量与标准

- 质量测量统一记录测量编号、编组、点位、数据类型、状态、时间和测量人。
- 橘皮、色差、光泽度、膜厚指标以标准化指标值存储，并关联适用标准。
- 标准支持工厂、车型、颜色、零件、点位、数据类型和有效期等适用范围。

### AI 与闭环

- 点位特征快照保存一次预测所使用的完整工艺、材料和上下文特征。
- 模型版本记录训练数据血缘、指标、制品地址、上线状态和适用范围。
- 预测、诊断、推荐、审批、执行和复测评价全程留痕。

## 3. 核心数据链

```text
生产事件
  -> 五段工艺阶段实绩
  -> 程序版本与刷子参数
  -> 刷子到测量点贡献权重
  -> 材料批次与实际参数
  -> 点位特征快照
  -> 质量测量与标准判定
  -> 预测 / 诊断 / 推荐
  -> 人工审批 / 新程序版本
  -> 执行 / 复测 / 效果评价
```

## 4. 实施阶段

### 阶段 0：工程基础与产品原型

当前已完成：

- Next.js 管理端、FastAPI 服务和 MySQL Docker 编排。
- 32 张核心领域表的 SQLAlchemy 模型与 Alembic 初始迁移。
- 工艺质量驾驶舱、程序中心、质量中心、AI 工作台和主数据中心界面。
- Dashboard、主数据、程序、质量、点位特征、预测、诊断、推荐和审批 API。
- 驾驶舱实时 API 数据读取、降级快照和真实审批 API 调用。
- 42 个工艺/材料参数定义目录、67 个质量指标目录和幂等演示数据。
- 基于刷子贡献权重、实际参数、材料批次和质量标签的点位特征快照。
- 可持久化岭回归基础模型训练、真实预测、相关性诊断和硬边界约束推荐。
- 推荐审批、执行值记录、复测质量关联和闭环效果评价。
- API Key 身份认证、标准角色权限目录、关键写操作审计与审计中心。

### 阶段 1：主数据与程序版本管理

- 完成工厂、车型、颜色、零件、编组、点位、程序、刷子表的完整 CRUD。
- 完善字段级权限、版本差异、封样和审批流程。
- 在实际 MySQL 测试环境执行迁移与回滚验证。
- 验收：可从界面建立一套可追溯的工厂、车型、颜色、程序和点位关系。

### 阶段 2：质量数据与点位聚合

- 接入 MES、机器人、材料和测量设备数据，建立可重放的数据导入任务。
- 实现刷子贡献权重维护和点位特征聚合服务。
- 实现质量标准判定、异常标记、缺失值和数据质量监控。
- 验收：任一质量测量结果均可追溯到生产事件、程序版本、材料批次和点位特征。

### 阶段 3：质量分析与规则诊断

- 上线 SPC、趋势、标准符合性、车型颜色对比和点位热力图。
- 建立基于工艺知识库的规则诊断和硬边界检查。
- 验收：异常点位可获得可解释的规则诊断及关联证据。

当前已完成质量标准自动匹配、上下限判定和超差原因输出；SPC、趋势和热力图仍待开发。

### 阶段 4：AI 模型与 MLOps

- 建立训练数据快照、特征版本、模型注册、离线评估和在线监控。
- 分质量指标训练预测模型；以 SHAP、敏感性分析和约束优化支持诊断与推荐。
- 设置车型、颜色、工厂适用范围及模型降级策略。
- 验收：模型达到约定精度，且每次结果均可追溯到模型版本和特征快照。

当前已完成用于验证链路的岭回归基础模型、模型持久化、预测记录和相关性贡献诊断；正式模型评估、漂移监控和模型制品存储仍待开发。

### 阶段 5：受控工艺闭环

- 推荐通过硬安全边界、TDS、单次调整步长和冲突校验。
- 审批后生成新程序版本并进入受控试验，不直接覆盖当前生效版本。
- 执行后自动关联复测结果，评价实际改善、偏差和副作用。
- 验收：完成“发现风险 -> 推荐 -> 审批 -> 执行 -> 复测 -> 评价”的审计闭环。

当前已完成硬边界约束推荐、审批、执行、复测和效果评价 API 及操作界面；MES、QMS、机器人和材料系统的幂等事件、失败重试、死信和重放框架已实现，真实现场地址、凭证与字段映射待配置。

### 阶段 6：生产部署与持续运营

- 对接企业 SSO，并完成备份、灾备、告警和可观测性。
- 前端部署至 Vercel 或企业 Web 平台，FastAPI、AI Worker 和 MySQL 部署在企业内网。
- 以工厂为单位灰度上线，持续监控模型漂移和闭环收益。

当前已完成 API Key 鉴权、RBAC 角色权限目录、写操作审计和审计中心；企业 SSO、
密钥托管与定期轮换仍需结合客户基础设施落地。

## 5. 关键验收指标

- 数据关联完整率、点位特征可用率和实际参数覆盖率。
- 各质量指标预测误差、风险召回率和误报率。
- 推荐通过率、审批率、真实改善率和平均闭环时长。
- 所有程序变更、模型结果和审批操作的审计可追溯率为 100%。
