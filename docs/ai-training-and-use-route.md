# PQ-AI AI 训练与现场使用路线

本文件描述当前代码已经实现并由测试约束的 AI 全生命周期。它用于开发、测试和工厂数据接入，不替代工厂模型验收结论。

## 1. 准备训练数据

两类来源地位相同，可单独使用，也可混合使用：

1. 生产记录：以“生产事件/车号 + 测量点”为一行，后端汇总五个喷涂阶段的刷子贡献、喷涂实绩、受治理材料结果和已验证质量结果。
2. 人工训练宽表：下载全中文 Excel/CSV 模板，填写样本编号、独立分组、样本时间、目标值和真实工艺/材料参数，上传前先校验。

系统只接受批准范围内的膜厚、色差/效应和橘皮目标。生产质量结果必须为 `VERIFIED`；人工文件必须通过列名、数据类型、时间、分组、重复样本和特征范围校验。

## 2. 生成不可变训练集

用户选择目标指标和数据来源后生成数据集快照。后端自动完成：

- 取所有样本共同存在的数值特征，形成训练宽表矩阵。
- 同一车身、生产事件、批次或人工试验分组不得跨训练集和验证集。
- 按样本时间把较新的完整分组留作最终验证，禁止随机拆点位行。
- 冻结来源记录、目标测量、特征值、分组、时间切点和泄漏检查结果。

人工样本和生产样本都会进入时间留出及独立分组验证；工厂、车型、颜色验证轴只统计具备生产上下文的样本，并明确记录被排除的人工样本数。

## 3. 自动训练与选型

默认选择“系统自动比较并选择”。系统只使用训练集内部的按时间扩展验证折比较：

- 稳健线性模型：保留全部参数影响，适合作为相关工艺参数较多时的稳定基线。
- 关键参数筛选模型：通过 Elastic Net 抑制弱影响参数，适合稀疏的小样本工业宽表。
- 多组防过拟合强度和参数筛选比例。

最终验证集不参与模型选择。选定方案在全部训练样本上重新拟合，再只在最终验证集上计算 MAE、RMSE、R² 和预测区间基线。模型载荷、数据集引用、指标和训练样本数一起计算 SHA-256 校验码并登记。

## 4. 人工验收与激活

模型训练后保持草稿状态，不能直接用于现场。激活前必须同时满足：

- 数据集泄漏检查通过，存在独立验证和多轴验证证据。
- 工件已登记且校验码一致。
- 每个适用工厂都有生效的目标指标验收策略，且 RMSE、R²、训练组数和验证组数达标。
- 工厂、车型、颜色适用范围已人工确认。
- 缺失参数和分布外输入阻断规则已人工确认。
- 最新人工验收结论为通过。

同一质量指标和同一工厂/车型/颜色范围激活新模型时，后端自动退役重叠的旧生效模型，避免现场调用结果不确定。

## 5. 点位预测、诊断和建议

在 2D/3D 车身点位页面选择生产记录并点击测量点。后端按当前工厂、车型、颜色、点位特征版本和输入分布筛选可用模型，前端只让用户选择可分析的中文质量指标。

- 预测：保存模型版本、点位特征快照、预测值、基于最终验证误差的参考区间、数据可信度和治理证据。
- 诊断：展示当前样本的参数贡献，只能标记为关联；DOE 或受控试验验证后才能确认原因。
- 参数建议：只调整已允许推荐且存在工厂批准硬边界来源的参数。用户不填写目标时，后端自动采用当前车型、颜色、零件、点位匹配度最高的生效质量标准。

建议不会直接修改机器人。它必须进入受控试验、不同人员审批、完整刷子表新版本、执行值记录、质量复测、效果评价和必要时回滚流程。

## 6. 运行与监控

- 在线调用只允许 `ACTIVE` 模型；范围外、参数缺失、分布外或治理配置失效立即阻断。
- 漂移报告比较训练分布、近期点位参数和有真实质量结果的在线误差。
- 当前训练在 API 请求内同步执行，适合开发验证和中小型工厂数据集。大规模生产训练仍需接入独立 AI Worker/任务队列。
- 当前模型制品保存在 MySQL 受治理载荷中。生产部署仍需接入企业对象存储/模型仓库、备份、监控告警和工厂真实验收数据。

主要入口：前端“AI 优化工作台”和车身点位详情；后端 `/api/ai/models/datasets`、`/train`、`/acceptance`、`/available`、`/{model_id}/predictions`、`/predictions/{id}/diagnoses`、`/{model_id}/recommendations`。
