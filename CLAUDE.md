# PQ-AI 项目规则与领域知识

## 项目定位
PQ-AI 是汽车涂装工艺与质量智能化闭环系统，面向涂装工艺工程师和质量工程师。

## 核心领域知识

### 1. 汽车涂装 3C2B 工艺流程
- **3C** = 中涂(Primer) + 色漆(Basecoat) + 清漆(Clearcoat) 三次喷涂
- **2B** = 中涂烘烤(140-150℃×30min) + 面漆烘烤(140-155℃×25-30min) 两次烘烤
- 完整工序链: 前处理→电泳(ED,膜厚20-25μm)→电泳打磨→中涂喷涂(20-40μm)→中涂烘烤→中涂打磨→色漆内板→色漆外板(8-22μm)→闪干(80℃×5-8min,脱水率≥92%)→清漆内板→清漆外板(30-45μm)→流平(10±1min)→面漆烘烤→检验
- 喷漆室环境: 温度21-25℃, 湿度60-70%RH, 风速0.3-0.4m/s
- 3C1B是3C2B的紧凑变体, 中涂不单独烘烤(湿碰湿), 节能但对材料要求更高

### 2. BYK 橘皮仪 (Wave-Scan) 数据结构
- 核心原理: 激光光学轮廓 → 带通滤波 → 结构光谱(6波段)
- 结构光谱: du(<0.1mm) → Wa(0.1-0.3mm) → Wb(0.3-1mm) → Wc(1-3mm) → Wd(3-10mm) → We(10-30mm)
- 聚合值: LW=1.2-12mm(Wc+Wd), SW=0.3-1.2mm(Wb+Wa), DOI=du+Wa+Wb的函数(ASTM E430)
- SW/LW平衡: 降低SW→更鲜亮但LW更明显; 降低LW→改善远观但高SW会遮盖LW
- OEM标度: BMW N1/N3, GM Tension/R-value, Honda P-Tension/H-Tension, VW/Audi LW/SW
- 色差仪(Spectro-Guide): CIE Lab*, 多角度(15°25°45°75°110°), dE*/dL*/da*/db*/dC*/dH*

### 3. Fischer 膜厚仪
- 磁感应法(ISO 2178): 非磁性涂层在钢/铁上 → 总膜厚
- 振幅涡流法(ISO 2360): 非导电涂层在铝/铜上 → 漆膜厚
- 相位涡流法(ISO 21968): 导电涂层(锌)在任意基材上, 可穿漆层测量
- Duplex双镀层: 总厚(磁感应) - 锌厚(相位涡流) = 漆厚
- PHASCOPE PMP10 DUPLEX 三合一, 自动基材识别
- 汽车参考膜厚: 总80-150μm, ED 15-25μm, 中涂20-40μm, 色漆8-22μm, 清漆30-50μm

### 4. Dürr 静电高压旋杯 (EcoBell)
- 雾化原理: 高速旋转(30k-70k rpm)离心力→杯缘液膜→破裂雾化
- 成型空气: 外成型空气(Outer)控制外边界, 内成型空气(Inner)控制内部雾流
- 静电包裹: 60-80kV高压电极充电→带电荷粒子被车身吸引→提高转移效率
- 五大核心参数: spray_flow(流量), bell_speed(转速), outer_air(外成型空气), inner_air(内成型空气), voltage(高压)
- 机器人(EcoRP): SPLINE(连续恒速,喷漆首选), LIN(直线), CIRC(圆弧), PTP(快速移动)
- 轨迹参数: 喷涂距离200-300mm, 速度400-1200mm/s, 重叠率50-75%(使膜厚标准差最小化)
- 编程工具: Dürr DXQ3D.onsite (离线编程+仿真+膜厚模拟+碰撞检查)

### 5. 涂料材料特性
- 固含-粘度-膜质量三角形: 低固含→膜收缩大→效应颜料定向好→VOC高; 高固含→施工窗口窄
- 效应颜料: 铝粉(3-25μm×0.1-2μm), 云母珠光, Flop Index翻转效应
- 关键风险: 清漆再溶解(Redissolving) → 清漆溶剂渗入色漆→破坏颜料定向
- 水性漆: 仅10-15%共溶剂, 需>230°F彻底脱水
- 温度敏感性: 漆温每升1℃→粘度降3-8%
- 膜缺陷关联: 橘皮(流平不足/粘度高), 针孔(溶剂挥发快/脱水不足), 流挂(粘度低/膜厚高), 色差(膜厚不均/批次差异)

### 6. 涂装工程师工作流程
- 工艺工程师: QC工程图→工艺卡→SOP→PFMEA→控制计划→工艺纪律检查→5W分析→优化
- 质量工程师: 在线巡检→终检(外观/膜厚/附着力)→性能检测→AUDIT→8D报告→鱼骨图→SPC
- 5M1E: 人(培训/技能)机(TPM)料(入厂检/批次追溯)法(工艺纪律/FMEA)环(洁净度/温湿度)测(仪器校准/SPC)
- CQI-12涂装系统评审: AIAG标准, 覆盖质量体系/原料/工艺/设备/环境/人员/监控/检验/环保
- 改善闭环: 发现问题→现地现物→5W真因→制定措施→实施→验证→标准化
- 核心理念: "有序+清洁=质量", 工艺稳定性=质量稳定性

## 项目技术规则

### 开发规则
- 前端零外部依赖: 仅Next.js+React+Lucide, 不引入状态管理库/CSS框架
- 所有API调用通过Next.js Route Handler反向代理, 附加x-api-key
- 后端FastAPI, SQLAlchemy 2风格, Pydantic v2
- 测试用SQLite内存库(pytest+httpx), 不依赖MySQL
- 所有写操作必须通过安全中间件审计(audit_log)
- 前端/后端/数据库/中间件/验证每个环节字段必须一致

### 代码提交前强制验证（强制）
所有代码变更提交前必须通过以下四项验证，未通过不得提交：

1. **自查**: Python AST语法通过、TypeScript类型无错误、导入路径正确、无调试代码残留
2. **测试**: `pytest tests/` 全部通过，新功能含测试用例
3. **构建**: `npm run build:web` TypeScript + Next.js 构建成功，Dockerfile COPY 路径存在
4. **用户交互模拟**: 页面可渲染、API可调用、表单/按钮操作可用、空/加载/错误状态完整

违反以上规则提交的代码将被回退并修复。

### 数据模型规则
- 主键统一UUID(36字符), 基础模型UUIDPrimaryKeyMixin+TimestampMixin
- 配置值(configured_value)与实绩值(actual_value)分表存储
- AI训练优先使用实绩值(actual_parameter), 配置值作为回退
- 工艺推荐必须经过硬安全边界检查+人工审批, 不允许自动修改机器人参数
- 点位特征版本化管理(feature_set_version), 模型血缘可追溯

### 质量模型规则
- 四种质量类型: ORANGE_PEEL(橘皮), COLOR_DIFFERENCE(色差), GLOSS(光泽度), THICKNESS(膜厚)
- 质量标准按优先级匹配: 点位级 > 零件级 > 颜色级 > 车型级 > 全局级
- 质量指标使用corrected_value(如有)或raw_value

### 工艺阶段规则
- 五工序: MIDCOAT_EXT → BASECOAT_1 → BASECOAT_2 → CLEARCOAT_1 → CLEARCOAT_2
- 一二站比例(basecoat_pass_ratio/clearcoat_pass_ratio)控制两站间分配
- 刷子贡献权重(BrushPointContribution)决定点位特征聚合时各刷子的影响力

## 已知边界与待完善项
- 基础模型(岭回归)仅验证链路, 非生产精度; 生产需升级XGBoost/LightGBM
- 当前无SPC控制图、膜厚热力图、趋势分析面板
- 缺少前处理/电泳/烘烤工序模型(仅覆盖3C, 缺2B和前处理)
- 无模型漂移监控、自动重训练
- 无移动端适配、大屏看板、离线模式
- 无8D报告工具、PFMEA维护、CQI-12审核模块
- 无真实MES/QMS/PLC凭证连接, 集成框架仅虚拟端点
